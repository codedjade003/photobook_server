🔓 FEATURE 4: 2FA SETUP (TOTP + BACKUP CODES)
==============================================

Step 1 - Initialize 2FA Setup
Test: POST /api/auth/2fa/setup
Auth: Bearer {JWT_TOKEN}
Headers: Authorization: Bearer YOUR_TOKEN

Expected Response:
{
  "message": "2FA setup initiated",
  "secret": "JBSWY3DPEBLW64TMMQ======",
  "qrCodeUrl": "data:image/png;base64,iVBORw0KG...",
  "backupCodes": ["ABCD1234", "EFGH5678", ...]
}

Action: Scan QR code with Google Authenticator, Authy, or Microsoft Authenticator


Step 2 - Confirm 2FA (After scanning QR code)
Get current TOTP token from authenticator app (6 digits)
Test: POST /api/auth/2fa/confirm
Auth: Bearer {JWT_TOKEN}
Body:
{
  "token": "123456",
  "secret": "JBSWY3DPEBLW64TMMQ======",
  "backupCodes": ["ABCD1234", "EFGH5678", ...]
}

Expected:
- 200 status
- Message: "2FA enabled successfully"
- User object with two_fa_enabled: true

Check DB:
- SELECT two_fa_enabled, two_fa_secret FROM users WHERE id = user_id
- Should show true and secret populated


Step 3 - Verify 2FA Code (Login scenario)
Test: POST /api/auth/2fa/verify
Body:
{
  "token": "123456"
}

Expected:
- 200 status
- { "success": true, "method": "totp" }


Step 4 - Verify with Backup Code
Test: POST /api/auth/2fa/verify
Body:
{
  "backupCode": "ABCD1234"
}

Expected:
- 200 status
- { "success": true, "method": "backup-code" }
- Check DB: backup code should be removed from two_fa_backup_codes array


Step 5 - Disable 2FA (Optional)
Test: DELETE /api/auth/2fa/disable
Auth: Bearer {JWT_TOKEN}

Expected:
- 200 status
- User object with two_fa_enabled: false

🐛 DEBUGGING TIPS
==================

If Email Not Received:
1. Check RESEND_API_KEY is correct in .env
2. Check EMAIL_FEATURE_ENABLED=true
3. Check template IDs are correct in .env
4. Check domain verified in Resend dashboard
5. Look at console logs for error messages
6. Check email spam folder
7. Check Resend dashboard for email history

If 2FA Not Working:
1. Verify JWT token in Authorization header is valid
2. Check authenticator app time is synced
3. Verify QR code was scanned into app
4. Check template installation in authenticator app
5. Try token from different time window (before/after 30 sec)

If Health Check Fails:
1. Check DATABASE_URL is correct
2. Check REDIS_URL is correct
3. Check both services are running
4. Look at console output for specific errors

If Google OAuth Fails:
1. Check GOOGLE_CLIENT_ID format matches
2. Check GOOGLE_CLIENT_SECRET is correct
3. Verify callback URI matches in Google Console
4. Check profile object has emails array
