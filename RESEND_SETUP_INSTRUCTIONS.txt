RESEND EMAIL DELIVERY SETUP GUIDE
=================================

This project now renders email templates in-house (local code) and uses Resend only to deliver emails.
No Resend template IDs are required.

Prerequisites:
- Resend account: https://resend.com
- Access to your DNS provider for domain verification

STEP 1: Verify Sending Domain
=============================

1. Go to https://resend.com/settings/domains
2. Add your domain (example: photobookhq.com)
3. Add DNS records provided by Resend
4. Wait until the domain status is Verified

STEP 2: Create API Key
======================

1. Go to https://resend.com/api-keys
2. Create an API key (example name: photobook-backend)
3. Copy the key (starts with re_)

STEP 3: Configure .env
======================

Required:

RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxxx
EMAIL_FROM=no-reply@photobookhq.com
EMAIL_FEATURE_ENABLED=true

Optional email behavior tuning:

VERIFICATION_CODE_TTL_MINUTES=15
VERIFICATION_MAX_ATTEMPTS=5
VERIFICATION_LOCKOUT_MINUTES=15
VERIFICATION_RESEND_MAX_ATTEMPTS=3
VERIFICATION_RESEND_WINDOW_MINUTES=60
VERIFICATION_RESEND_COOLDOWN_SECONDS=60
PASSWORD_RESET_CODE_TTL_MINUTES=15

STEP 4: Restart API
===================

Restart the backend so the environment is reloaded:
- npm run dev

STEP 5: Test Email Flows
========================

A) Signup verification email
- POST /api/auth/signup
- Expect: account created, verification code email delivered

B) Resend verification code
- POST /api/auth/verify-email/resend
- Expect: new code email delivered (subject to cooldown and limits)

C) Password reset email
- POST /api/auth/password-reset/request
- Expect: reset code email delivered

TROUBLESHOOTING
===============

If you get email delivery errors:
1. Confirm RESEND_API_KEY is valid
2. Confirm EMAIL_FROM domain/address is verified in Resend
3. Confirm EMAIL_FEATURE_ENABLED=true
4. Restart server after env changes
5. Check backend logs for provider error details

Notes:
- Local templates live in src/templates/email.templates.js
- Resend templates and template IDs are not used by this backend anymore
